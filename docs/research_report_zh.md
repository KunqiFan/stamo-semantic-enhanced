# StaMo Process Bridge 研究成果总报告

**项目**：基于紧凑过程语义的机器人操作状态表示学习  
**日期**：2026-05-07（更新）  
**数据集**：DROID（3,297 样本 = 2,659 train + 638 test，按轨迹划分，70/15 episodes，无重叠）

---

## 摘要

机器人操作的策略学习依赖于对当前状态的理解——"夹爪是否接触物体？物体在移动还是静止？离目标更近了吗？"这些操作语义是决策的基础。视觉条件扩散模型（如 StaMo）能从当前帧生成下一帧，其内部的压缩表示（delta_z）理论上编码了帧间状态转移的全部信息。如果这些表示同时承载语义，就能为下游策略提供一个紧凑、统一的状态接口——既服务于视觉生成，又服务于语义决策。

然而，StaMo 的压缩流水线（DINOv2 → Projector → DiTConditionHead，384:1）完全由扩散生成损失驱动。我们发现，这种单一目标驱动的压缩**系统性地丢弃了语义信息**：纯扩散训练的 delta_z 语义分类准确率仅 ~54%（接近随机基线），而压缩前的 DINOv2 原始特征高达 ~97%。语义信息在压缩入口处充分存在，但因对像素重建贡献微弱而被 Projector 丢弃。这意味着，如果不加干预，扩散模型的潜在表示无法直接用于语义层面的状态理解——生成能力和语义理解被割裂在两个独立的表示空间中。

为弥合这一割裂，我们提出在压缩流水线中并行添加轻量级语义分类头（SemanticHead，~0.6M 参数，占总可训练参数 0.6%），通过多任务学习改变 Projector 的压缩策略。结果表明，delta_z 的语义分类性能从 ~54% 提升至 95.9%（contact_state），几乎追平 DINOv2 原始特征，同时扩散生成质量不变（PSNR 9.10→8.96，差异在统计误差内）。一套表示同时服务两个目标，语义理解不再需要独立的特征提取管道。

两组消融实验揭示了内在机制：（1）λ 权重消融发现语义分类性能呈 U 型曲线（λ=0.1 最优 72.9%，λ=2.0 最差 56.9%），表明扩散与语义目标之间存在梯度竞争，等权组合是最差选择；（2）SemanticHead 接入位置消融发现 Projector 输出处（Position B, 68.8%）显著优于 DiTConditionHead 输出处（Position A, 60.9%）和 DINOv2 原始特征处（Position C, 53.7%）。B > C 表明 Projector 在语义梯度驱动下不仅保留了语义，还主动放大了语义编码能力，超越了输入特征本身。关键变量不是压缩比，而是语义梯度能否有效到达 Projector。

下游验证表明，从语义分类头提取的 12 维紧凑语义向量在阶段分类（78%）、few-shot 学习（2% 数据达 91.7%）和动作预测（z 轴 R²=0.76）等任务中均显著优于 512 维 delta_z（信息压缩比 42.7:1），初步验证了语义感知的压缩表示作为下游策略状态接口的可行性。

## 一、研究背景与核心问题

### 1.1 研究动机

机器人操作（manipulation）的过程理解要求系统能够从视觉观测中推断当前的操作语义状态。现有视觉-语言-动作（VLA）模型往往直接从高维视觉特征映射到动作空间，缺乏对操作过程中间状态的显式建模。

本研究提出**紧凑过程语义（Compact Process Semantics）**作为统一的语义接口，定义了 4 个离散属性：

| 语义字段 | 类别数 | 标签 | 物理含义 |
|---------|-------|------|---------|
| contact_state | 2 | no_contact, contact | 夹爪是否接触物体 |
| gripper_state | 3 | open, closing, closed | 夹爪当前状态 |
| object_motion | 4 | still, lifted, moved, placed | 被操作物体的运动状态 |
| target_relation | 3 | farther, closer, reached | 末端执行器与目标的相对距离变化 |

### 1.2 核心研究问题

StaMo 的视觉压缩流水线（DINOv2 → Projector → DiTConditionHead）将 196,608 维的视觉特征压缩到 512 维，压缩比高达 384:1。关键在于：**压缩本身不是问题，问题是谁在决定压缩的方向。**

当 Projector 只被扩散损失训练时，它学到的压缩策略是"保留对像素重建有用的一切，丢弃其余"。语义信息——比如"夹爪是否接触物体"——对逐像素重建的贡献微弱，因此被选择性丢弃。这不是压缩比太大的问题（DINOv2 自身也做了大规模压缩，语义保留得很好），而是**单一任务目标垄断了压缩方向**的问题。

核心问题是：

1. **任务驱动压缩的语义代价**：扩散目标单方面驱动的压缩，选择性丢弃了多少语义信息？丢在哪一级？
2. **多目标压缩重定向**：能否通过注入轻量级语义监督，在不损害生成能力的前提下，改变压缩策略使其同时保留语义？
3. **状态抽象价值**：被"救回"的语义信息能否作为有效的状态抽象用于下游任务？

### 1.3 任务驱动压缩的选择性丢弃机制

理解本研究的所有实验结果，需要先理解一个核心机制：**压缩是中性的，优化目标决定压缩丢弃什么。**

#### 信息瓶颈中的竞争

Projector 将 256×768 = 196,608 维压缩到 2×1024 = 2,048 维（96:1）。在这个极端瓶颈中，不可能保留所有信息——Projector 必须做选择。选择的依据是：**什么信息能最有效地降低损失函数？**

```
DINOv2 输出 [256×768]
  包含：纹理、边缘、物体形状、空间关系、语义类别、光照...
         ↓
    Projector (96:1 压缩)
    选择依据 = ∂Loss/∂representation
         ↓
  保留的信息取决于 Loss 是什么：

  Loss = 扩散MSE     → 保留纹理、边缘、空间布局（像素重建需要）
                       丢弃语义类别、接触状态（对重建无用）

  Loss = 扩散 + 语义  → 保留纹理 + 语义类别 + 接触状态
                       丢弃对两者都无用的信息
```

这解释了为什么：
- **实验一**（纯扩散）：delta_z 无语义（~54%）——Projector 没有理由保留语义
- **实验三**（扩散+语义）：delta_z 有语义（95.9%）——语义损失的梯度驱动 Projector 主动编码语义
- **生成质量不变**（PSNR 9.10→8.96）——语义信息和像素信息大部分不冲突，Projector 有足够容量同时编码两者

#### 为什么语义对像素重建"无用"

这不是显然的。直觉上，"知道夹爪正在接触物体"似乎有助于预测下一帧画面。但扩散模型的损失函数是**逐像素的 MSE/flow matching**——它不需要理解"接触"这个概念，只需要知道"这些像素的颜色值应该是多少"。

具体来说：
- "contact_state = contact" 意味着夹爪和物体在画面中重叠——但 Projector 已经通过编码空间位置信息（哪些像素在哪里）隐式地捕获了这一点，不需要显式的"接触"概念
- "gripper_state = closing" 意味着夹爪间距在缩小——同样，像素级的运动信息已经足够，不需要"关闭"这个语义标签
- 换言之，扩散模型用**低层视觉特征**（位置、运动、纹理）就能完成重建，不需要**高层语义概念**

这正是信息瓶颈理论的预测：当下游任务只需要低层信息时，高层语义在压缩中会被优先丢弃，因为它们占用表示容量却不降低损失。

#### 多任务学习如何改变压缩方向

添加语义损失后，Projector 的梯度来自两个方向：

```
                    Projector 参数 θ
                   ╱              ╲
          ∂L_diff/∂θ            ∂L_sem/∂θ
         "保留像素信息"         "保留语义信息"
```

两组梯度共同决定 Projector 的更新方向。只要两个任务的信息需求**不完全冲突**（即不存在"保留 A 必须丢弃 B"的零和博弈），Projector 就能找到一个同时服务两者的压缩策略。

实验证据支持"不完全冲突"的假设：
- 生成质量几乎不变（PSNR 9.10→8.96），说明语义信息的加入没有挤占像素信息的容量
- 但 λ 消融的 U 型曲线表明，当 λ 处于中间值时，两组梯度的**方向冲突**会导致优化困难——不是容量不够，而是梯度互相干扰

#### 本研究各实验在这个框架下的位置

| 实验 | 回答的问题 |
|------|-----------|
| 实验一（纯扩散） | 扩散目标单独驱动的压缩丢弃了语义吗？→ **是** |
| 实验二（DINOv2） | 压缩前的原始特征有语义吗？→ **有**（语义在入口处存在） |
| 实验三（并行训练） | 注入语义目标能改变压缩方向吗？→ **能**，且不损害生成 |
| λ 消融 | 两个目标的梯度如何竞争？→ U 型曲线，等权最差 |
| 位置消融 | 语义在压缩流水线的哪一级被丢弃？→ 逐级量化 |

### 1.4 标签来源

语义标签通过启发式规则从机械臂传感器数据自动标注：

- **contact_state**：基于夹爪开合度阈值判定
- **gripper_state**：基于夹爪位置的帧间差分方向
- **object_motion**：基于末端执行器位移幅度 + Z 轴方向变化
- **target_relation**：基于局部速度方向 + 目标距离变化趋势

这种自动标注方式避免了人工标注的成本，但引入了标注噪声（尤其 target_relation 的启发式精度有限）。

## 二、系统架构详解

### 2.1 整体架构

```
原始图像 (224×224×3)
    ↓
VisionBackbone: DINOv2 ViT-B/14 (冻结，86.4M参数)
    ↓
特征图 [batch, 256, 768]    ← 256 个 patch token，每个 768 维
    ↓
Projector (可训练，交叉注意力 ×4层)
    ↓
压缩表示 [batch, 2, 1024]   ← 从 256 个 token 压缩到 2 个 token (delta_z)
    ↓
DiTConditionHead (可训练)
    ↓
pooled_embeds [batch, 512]   ← 进一步池化为单一向量 (delta_pooled)
    ├── DiT Transformer → 扩散生成 → diffusion_loss
    └── SemanticHead → 语义分类 → semantic_loss
```

信息压缩链条：

| 层 | 维度 | 压缩比 |
|----|------|-------|
| 原始像素 | 224×224×3 = 150,528 | 1× |
| DINOv2 输出 | 256×768 = 196,608 | 1.3× (特征提取，维度略增) |
| Projector 输出 (delta_z) | 2×1024 = 2,048 | 96× |
| DiTConditionHead 输出 (delta_pooled) | 512 | 384× |
| 语义预测 | 12 (one-hot) | 16,384× |

### 2.2 各模块详解

#### 2.2.1 VisionBackbone: DINOv2 ViT-B/14

DINOv2 是 Meta 使用自监督对比学习训练的视觉 Transformer。训练目标不依赖任何人工标签，纯靠"同一张图的不同随机裁切（augmented views）应该有相似的特征表示"这一自监督信号。

**关键性质**：DINOv2 的特征天然具有语义区分能力——能区分物体边界、姿态、空间关系等。这是 DINOv2 作为 VisionBackbone 的理论基础。

在本系统中 DINOv2 被**完全冻结**（86.4M 参数不参与训练），作为固定的通用视觉特征提取器。ViT-B/14 表示使用 Base 规模模型、14×14 的 patch 大小，输入 224×224 图像产出 (224/14)² = 256 个 patch token。

#### 2.2.2 Projector: 交叉注意力压缩器

Projector 的作用是信息瓶颈——将 DINOv2 的 256 个 patch token 压缩为 2 个 learnable query token。

架构：4 层交叉注意力（Cross-Attention），每层中 2 个可学习的 query 向量 attend to 256 个 DINOv2 patch token。直觉上，Projector 在学习"从 256 个局部视觉特征中，提取出最重要的 2 个全局描述子"。

**信息论角度**：压缩比 96:1 意味着不可避免地丢弃大量信息。Projector 丢什么、留什么，完全取决于损失函数提供的梯度信号。这正是本研究的核心观测——不同的损失函数（纯扩散 vs 扩散+语义）会导致 Projector 学到根本不同的表示。

#### 2.2.3 DiTConditionHead: 池化 + 线性投影

DiTConditionHead 将 Projector 的 2-token 输出进一步压缩为单一向量：

```python
def forward(self, inputs):           # inputs: [batch, 2, 1024]
    pooled = inputs.mean(dim=1)      # Step 1: 平均池化 → [batch, 1024]
    pooled = self.pooled_align_mlp(pooled)  # Step 2: 线性投影 → [batch, 512]
    return pooled
```

只有两步操作：
1. **平均池化**：将 2 个 token 逐维取平均，压成 1 个 1024 维向量
2. **线性投影**：矩阵乘法 (1024→512)，将维度映射到 DiT 的 pooled condition 空间

DiT 需要两种条件输入：
- **image_embeds [2, 1024]**：作为 cross-attention 的 key/value，传递局部细节
- **pooled_embeds [512]**：作为全局条件，拼入 timestep embedding，控制生成的全局风格和布局

#### 2.2.4 DiT: 扩散生成器

本研究使用从零训练的小型 DiT（Toy DiT），而非预训练的 Stable Diffusion 3：

| 参数 | 值 |
|------|-----|
| 层数 | 6 |
| 注意力头数 | 8 |
| 注意力头维度 | 64 |
| 潜在空间大小 | 56×56 |
| 训练时间步 | 1000 |
| 推理步数 | 20 (DDPM) |

**扩散模型的核心思想**：
1. **前向过程**：给干净图像逐步加高斯噪声，经过 T 步后变为纯噪声
2. **反向过程**：训练神经网络学会"去噪"——给定噪声图和时间步 t，预测所加的噪声
3. **推理**：从纯噪声出发，迭代去噪 20 步，生成图像

本系统使用 SD3 的 **Flow Matching** 变体：不是离散去噪，而是学习一个连续速度场，将噪声分布"流动"到数据分布。训练损失是预测速度场与真实速度场之间的 MSE。

**条件生成**：给定 image_t 的特征作为条件，DiT 生成 image_tp（下一帧）。这迫使 Projector 编码足够的视觉信息来"指导"生成过程。

#### 2.2.5 SemanticHead: 语义分类分支

SemanticHead 是本研究的核心创新——在 DiTConditionHead 输出后并行添加的轻量级分类分支：

```python
class SemanticHead(nn.Module):
    def __init__(self, input_dim=512, hidden_dim=256):
        self.heads = {
            "contact_state":  MLP(512 → 256 → 2),
            "gripper_state":  MLP(512 → 256 → 3),
            "object_motion":  MLP(512 → 256 → 4),
            "target_relation": MLP(512 → 256 → 3),
        }
        # 每个 MLP: Linear → ReLU → Dropout(0.1) → Linear

    def forward(self, delta_pooled):  # delta_pooled: [batch, 512]
        return {field: head(delta_pooled) for field, head in self.heads.items()}
```

- 参数量：~0.6M（对比 RenderNet 总体 106.7M 可训练参数，占比不到 0.6%）
- 输入：`delta_pooled = pooled_tp - pooled_t`（两帧 pooled 表示的差值）
- 输出：4 个字段各自的 logits，用 CrossEntropy 损失训练

**为什么用 delta_pooled 而非 delta_z**：delta_pooled (512D) 是 delta_z (2048D) 经过 DiTConditionHead 池化后的全局概要。语义标签描述的是全局状态转移（"夹爪碰到物体了吗？"），不需要 token 级的局部细节，用压缩后的全局向量更自然。但位置消融实验（§13）表明，这一步压缩会丢失部分语义信息——在 Projector 输出处（1024D）接入 SemanticHead 效果更好。

### 2.3 损失函数设计

```
total_loss = diffusion_loss + λ × semantic_loss
```

其中：
- **diffusion_loss**：DiT 预测噪声/速度场的 MSE 损失
- **semantic_loss**：4 个分类头的交叉熵损失取平均
- **λ**：权衡系数，控制语义监督的强度

λ 的选择决定了 Projector 的学习方向：
- λ = 0：纯扩散训练，Projector 只编码像素级信息
- λ → ∞：纯语义训练，Projector 只编码语义信息
- λ = 1.0：两者等权（默认配置）

### 2.4 训练中的图像对处理

语义并行训练使用图像对 (image_t, image_tp) 作为输入：

```python
# 将两帧 stack 后一次性通过 DINOv2 + Projector（节省计算）
stacked = torch.cat([image_t, image_tp], dim=0)     # [2B, 3, 224, 224]
all_embeddings, all_pooled = model.encode(stacked)    # [2B, 2, 1024], [2B, 512]

# 拆分
pooled_t, pooled_tp = all_pooled.chunk(2, dim=0)      # 各 [B, 512]
delta_pooled = pooled_tp - pooled_t                     # [B, 512]

# 两条路径并行
semantic_logits = semantic_head(delta_pooled)            # 语义分类
diffusion_loss = dit_forward(image_t_features, image_tp) # 扩散生成
```

这种 stack 优化使得 DINOv2 和 Projector 对两帧图像只做一次前向传播（而非两次），峰值 VRAM 仅从纯扩散的 2.60 GB 增加到 2.86 GB（+10%）。

## 三、评估方法论

### 3.1 评估流水线

评估分为两个完全独立的阶段：

```
阶段 1（GPU）：特征提取
  对测试集每个图像对：
    image_t, image_tp → DINOv2 → Projector → delta_z, delta_pooled
    保存为 .npz 文件

阶段 2（CPU）：分类评估
  加载所有 .npz 特征 + manifest 中的本体感觉信号
  → 组装特征向量 (2629D)
  → HistGradientBoostingClassifier 训练 + 评估
  → 输出 eval accuracy
```

### 3.2 特征组成 (2629D)

| 特征 | 维度 | 来源 | 是否受 λ 影响 |
|------|:----:|------|:----:|
| delta_z | 2048D | Projector 帧间差 | ✓ |
| delta_pooled | 512D | DiTConditionHead 帧间差 | ✓ |
| delta_z 统计特征 | 14D | 各 token 的均值/标准差/最大值/分位数/范数/正值比 | ✓ |
| delta_z top-k 激活 | 32D | 绝对值最大的 32 个维度的原始值 | ✓ |
| 本体感觉信号 | 23D | action(7D) + ee_delta(6D) + 派生特征(10D) | ✗ |

派生本体感觉特征包括：gripper_cmd, xyz_norm, z_component, rot_norm, action_xyz_norm, gripper_is_closing, gripper_is_opening, low_movement, grasp_signal, place_signal。

### 3.3 为什么用 sklearn 而非 SemanticHead 直接评估

| 评估方式 | 衡量的对象 | 优缺点 |
|---------|-----------|--------|
| SemanticHead 直接分类 | SemanticHead MLP 的能力 | 会过拟合训练集，eval loss 上升但 accuracy 可能虚高 |
| **sklearn 分类 delta_z** | **Projector 表示的语义可分性** | 独立分类器，不受训练过拟合影响，公平对比不同配置 |

核心原则：**评估的是 Projector 学到的表示质量**（delta_z 是否编码了足够的语义信息），而非某个特定分类头的性能。使用独立的 sklearn 分类器作为"标准化考官"，确保不同训练配置（不同 λ、不同步数）之间的对比是公平的。

### 3.4 HistGradientBoostingClassifier

选择梯度提升树（而非 MLP 等神经网络）作为分类器的原因：

| 特性 | 梯度提升树 | 神经网络 MLP |
|------|-----------|-------------|
| 训练时间 | 秒级 | 分钟级 |
| 样本量要求 | 几百即可 | 通常需要几千+ |
| 调参需求 | 少（默认参数即可工作） | 多（LR, 架构, dropout, epochs...） |
| 过拟合风险 | 低（内置正则化） | 高 |
| 可解释性 | 可查看特征重要度 | 黑盒 |

在仅 2659 个训练样本的条件下，梯度提升树是最稳定、可重复的选择。

超参数配置：
- max_iter=500（500 棵决策树）
- max_depth=6（每棵树最大 6 层）
- learning_rate=0.03（每棵树的贡献权重）
- class_weight="balanced"（自动处理类别不平衡）
- l2_regularization=0.1（L2 正则化防止过拟合）

### 3.5 生成质量评估指标

| 指标 | 全称 | 范围 | 越高/低越好 | 衡量的内容 |
|------|-----|------|-----------|-----------|
| PSNR | Peak Signal-to-Noise Ratio | 0~∞ dB | 越高越好 | 像素级重建精度 |
| SSIM | Structural Similarity Index | 0~1 | 越高越好 | 结构/纹理相似度 |
| LPIPS | Learned Perceptual Image Patch Similarity | 0~∞ | 越低越好 | 感知距离（用 AlexNet 特征计算） |

PSNR/SSIM 是传统信号处理指标，LPIPS 是基于深度网络的感知指标。两类指标有时会不一致（PSNR 高但 LPIPS 差），因为人类视觉对结构比对像素更敏感。

本研究经历了三个阶段，逐步深入：

```
阶段 1：纯扩散训练 → delta_z 语义分类（失败 → 负面结果）
```

## 五、实验一：纯扩散训练的 delta_z（阶段 1）

### 5.1 StaMo 模型训练

在 DROID 数据集上训练 StaMo（DINOv2 ViT-B/14 骨干 + Toy DiT 扩散头），5000 步，loss 从 1.42 降至 0.14。

- 架构：VisionBackbone (frozen) → Projector → DiTConditionHead → SD3 DiT
- 训练目标：纯扩散损失（像素级重建），λ=0
- 峰值 VRAM：2.60 GB

### 5.2 delta_z 语义分类结果

使用 HistGradientBoostingClassifier + 本体感觉特征（action 7D + ee_delta 6D + 派生特征），5 折交叉验证：

| 语义字段 | Eval Accuracy | 说明 |
|---------|--------------|------|
| contact_state | ~54% | 仅略高于多数类基线 |
| gripper_state | ~50% | 接近随机猜测 |
| object_motion | ~85% | 主要靠本体感觉信号，非 delta_z |
| target_relation | ~45% | 低于随机基线 |

### 5.3 负面结论

**扩散目标单方面驱动的压缩，选择性丢弃了语义信息。** 这不是因为 96:1 的压缩比"太狠"——DINOv2 自身也做了大规模压缩（150K 像素 → 196K 维特征），语义保留得很好（实验二将证明这一点）。问题在于扩散损失的梯度只奖励像素重建，Projector 没有任何激励去保留语义信息。证据链：
- delta_z 在 few-shot 设置下完全无法学习（所有数据量下 F1 ≈ 18%）
- 将 delta_z 加入其他特征后性能反而下降（引入噪声）
- 这是**优化目标垄断压缩方向**的必然结果——扩散损失的梯度只关心"重建像素准不准"，Projector 学到的压缩策略系统性地排除了语义信息

**与实验二的对照**：DINOv2 的自监督对比学习目标（"同一张图的不同裁切应有相似表示"）天然奖励语义一致性，因此其压缩保留了语义。同样的视觉输入，不同的优化目标，产生了截然不同的压缩结果。这直接支持了§1.3 的核心论点：压缩是中性的，优化目标决定压缩丢弃什么。

## 六、实验二：DINOv2 原始特征（阶段 2）

### 6.1 特征提取

使用 DINOv2 ViT-B/14 的 CLS token（768D），对每个图像对提取：
- feat_t：当前帧 CLS token（768D）
- feat_delta：帧间差异 = feat_tp - feat_t（768D）
- 本体感觉特征（23D）
- 总特征维度：1,559D

### 6.2 语义分类结果

| 语义字段 | DINOv2 (v4) Eval Acc | vs 纯扩散 delta_z | 提升 |
|---------|---------------------|------------------|------|
| contact_state | **96.9%** | 54% | +42.9pp |
| gripper_state | **91.7%** | 50% | +41.7pp |
| object_motion | **99.4%** | 85% | +14.4pp |
| target_relation | **66.8%** | 45% | +21.8pp |

### 6.3 混合特征实验（DINOv2 + delta_z）

| 语义字段 | DINOv2 only | DINOv2 + delta_z |
|---------|-------------|--------------------|
| contact_state | **96.9%** | 96.7% (-0.2) |
| gripper_state | **91.7%** | 91.2% (-0.5) |
| object_motion | **99.4%** | 99.3% (-0.1) |
| target_relation | **66.8%** | 65.5% (-1.3) |

**结论**：添加纯扩散 delta_z 不仅无益，反而引入噪声。这进一步证实了实验一的发现——扩散驱动的压缩不仅丢弃了语义，还用像素重建的"残渣"填充了表示空间，对语义分类构成干扰。

**本实验的角色**：证明语义信息在压缩流水线的**入口处**是充分存在的。DINOv2 的自监督目标（对比学习）天然奖励语义一致性，因此其 768D 输出保留了丰富的语义。问题不在于输入缺乏语义，而在于 Projector 的扩散驱动压缩将其丢弃。阶段 3 的目标是通过改变优化目标，让 Projector 在压缩时主动编码这些语义。

## 七、实验三：扩散 + 语义并行训练（阶段 3 — 核心贡献）

### 7.1 核心思想：改变压缩方向

实验一和实验二共同揭示了问题的本质：语义信息在 DINOv2 输出中充分存在（96.9%），但被扩散驱动的 Projector 压缩丢弃（~54%）。解决方案不是改变压缩比或架构，而是**改变优化目标**——在扩散损失之外添加语义损失，让 Projector 的梯度同时来自两个方向，使其在压缩时主动编码语义信息。

这是多任务学习（Multi-Task Learning）的经典应用：当两个任务共享底层表示（Projector）时，梯度从两侧同时回传，Projector 必须找到一个同时服务两个目标的压缩策略。

### 7.2 架构设计

在 DiTConditionHead 输出后添加 SemanticHead，与扩散路径并行：

```
VisionBackbone (frozen) → [bs, 256, 768]
  → Projector (trainable) → [bs, 2, 1024]      ← delta_z
    → DiTConditionHead → pooled_embeds [bs, 512] ← delta_pooled
      ├── DiT: diffusion loss（条件=image_t，目标=image_tp）
      └── SemanticHead: 4× CrossEntropy loss（输入=delta_pooled）
```

- SemanticHead：4 个独立 MLP 分类头（512→256→num_classes），参数量 ~0.6M
- 训练数据：图像对 (image_t, image_tp) + 语义标签
- 损失函数：total_loss = diffusion_loss + λ × semantic_loss（λ=1.0）
- VRAM 优化：将图像对 stack 后一次前向，峰值 2.86 GB

### 7.3 训练过程

| 指标 | 初始 (step 10) | 最终 (step 5000) |
|------|---------------|-----------------|
| total_loss | 2.41 | 0.98 |
| diffusion_loss | 1.32 | 0.18 |
| semantic_loss | 1.09 | 0.77 |
| 峰值 VRAM | 2.86 GB | 2.86 GB |
| 训练速度 | 1.3 step/s | 1.5 step/s |

扩散损失收敛到与纯扩散训练相同水平（0.14 vs 0.18），说明语义监督未显著损害生成能力。

### 7.4 新 delta_z 语义分类结果

| 语义字段 | 纯扩散 delta_z | 语义并行 delta_z | 提升 |
|---------|---------------|-----------------|------|
| contact_state | ~54% | **95.9%** | **+41.9pp** |
| gripper_state | ~50% | **91.5%** | **+41.5pp** |
| object_motion | ~85% | **99.4%** | **+14.4pp** |
| target_relation | ~45% | **64.6%** | **+19.6pp** |

### 7.5 与 DINOv2 原始特征的对比

| 语义字段 | DINOv2 (1559D) | 语义并行 delta_z (2629D) | 差距 |
|---------|---------------|------------------------|------|
| contact_state | 96.9% | 95.9% | -1.0pp |
| gripper_state | 91.7% | 91.5% | -0.2pp |
| object_motion | 99.4% | 99.4% | 0 |
| target_relation | 66.8% | 64.6% | -2.2pp |

**核心发现**：语义并行训练的 delta_z 几乎追平 DINOv2 原始特征（差距 0-2.2pp），同时保留了扩散生成能力。这证明多任务学习可以让经过 96:1 压缩的表示同时编码像素级变化和语义状态转移。

## 八、生成质量评估

### 8.1 评估方法

在测试集（638 样本）上逐样本对比条件生成质量：
- 输入：image_t（条件图像）
- 过程：eval_step() → 20 步 DDPM 去噪 → 生成 image_tp
- 对比：生成的 image_tp vs 真实 image_tp
- 指标：PSNR ↑ / SSIM ↑ / LPIPS ↓

### 8.2 结果

| 指标 | 纯扩散 (5000步) | 语义并行 (5000步) | 差异 | 结论 |
|------|----------------|------------------|------|------|
| **PSNR** ↑ | 9.1034 ± 1.1199 | 8.9647 ± 0.8804 | −0.14 (−1.5%) | 标准差重叠，差异不显著 |
| **SSIM** ↑ | 0.0575 ± 0.0119 | 0.0516 ± 0.0099 | −0.006 (−10.3%) | 绝对值差异极小 |
| **LPIPS** ↓ | 1.2557 ± 0.0281 | **1.2299** ± 0.0313 | −0.026 (−2.1%) | 语义并行感知质量略优 |

### 8.3 结论

添加语义并行分支（~0.6M 参数，占总可训练参数的 0.6%）未对扩散生成质量造成可观测的负面影响。LPIPS 感知距离反而略低（更好），与 PSNR/SSIM 的微降综合来看，两个模型的生成能力在统计意义上不可区分。这验证了语义分支作为轻量级旁路的"免费午餐"设计——以极小的额外计算开销获得语义能力，且不损害原有的生成能力。

注：两个模型的绝对生成质量均处于早期水平（PSNR ~9, SSIM ~0.05），符合 5000 步 Toy DiT 从零训练的预期。使用预训练 SD3 权重可大幅提升绝对质量。

## 十一、延续训练实验（5000→14000 步）

### 11.1 动机
5000 步时 semantic_loss（0.77）远高于 diffusion_loss（0.18），假设继续训练可进一步提升语义分类。

### 11.2 训练过程
从 5000 步 checkpoint 恢复训练，每 2000 步保存 checkpoint 并评估。

| Step | diffusion_loss | semantic_loss (train) | semantic_loss (eval) |
|------|:-:|:-:|:-:|
| 5000 | 0.18 | 0.77 | 0.93 |
| 8000 | 0.18 | 0.25 | — |
| 10000 | 0.18 | 0.15 | 1.99 |
| 12000 | 0.18 | 0.09 | **2.64** |
| 14000 | 0.18 | 0.07 | **2.37** |

### 11.3 过拟合分析

**过拟合信号明确**：train semantic_loss 从 0.77 降至 0.07（↓91%），但 eval semantic_loss 从 0.93 升至 2.64（↑184%）。

过拟合的机制：SemanticHead 在训练集上学会了"极端自信"的预测——对正确样本给出近 1.0 的概率（loss→0），但对测试集的错误样本同样给出极端自信的错误概率，导致交叉熵损失爆炸（-ln(0.02) ≈ 3.9）。

关键区分：eval accuracy 仍在微弱上升（52.9%→56.4%），但 eval loss 大幅上升。这是因为 accuracy 只看 argmax 是否正确，而 loss 考虑了概率分布的质量。模型变得"自信但不可靠"。

### 11.4 sklearn 分类对比

| 语义字段 | 5000步 Eval Acc | 14000步 Eval Acc | 变化 |
|---------|:-:|:-:|:-:|
| contact_state | **95.9%** | 95.1% | −0.8pp |
| gripper_state | 91.5% | **92.2%** | +0.7pp |
| object_motion | **99.4%** | 99.2% | −0.2pp |
| target_relation | **64.6%** | 50.0% | **−14.6pp** ⚠️ |

### 11.5 结论
**5000 步是当前架构和数据量下的最优训练长度。** 继续训练不仅无提升，反而因过拟合损害了 target_relation 的表示质量（Projector 的表示空间被 SemanticHead 的过拟合梯度扭曲）。

**启示**：进一步改进需要的不是更多训练步数，而是更多数据（减轻过拟合）、更强正则化（dropout 从 0.1 增大到 0.3）、或更强架构（预训练 SD3 替代 Toy DiT）。

## 十二、消融实验：λ 权重敏感性分析

### 12.1 实验动机

实验三证明了语义监督能改变压缩流水线的编码策略。但 λ 控制着语义监督的强度——它决定了 Projector 在"为扩散编码像素信息"和"为语义编码分类信息"之间如何分配表示容量。λ 的选择是否敏感？是否存在最优平衡点？

### 12.2 消融实验方法论

消融实验（Ablation Study）通过每次只改变一个变量、保持其他一切不变，来理解每个设计选择的贡献。与超参数调优不同，消融的目的不是找最优配置，而是理解机制和验证鲁棒性。

本消融实验固定所有条件（数据集、架构、种子、训练步数），仅改变 λ：

| 实验条件 | 值 |
|---------|-----|
| 数据集 | DROID 3297 样本（固定划分） |
| 架构 | DINOv2 + Projector + DiT + SemanticHead |
| 随机种子 | 33 |
| 训练步数 | 5000 |
| 学习率 | 5e-5 |
| 变量：λ | 0.1, 0.5, 1.0*, 2.0, 5.0 |

λ 候选值按对数尺度选取（覆盖从 0.1 到 5.0 的 50 倍范围），这是超参数搜索的标准做法。

### 12.3 λ 的物理含义

```
total_loss = diffusion_loss + λ × semantic_loss

λ = 0    → 纯扩散（阶段1实验）
λ = 0.1  → 语义信号很弱：Projector 几乎只被像素重建驱动
λ = 0.5  → 语义辅助扩散
λ = 1.0  → 两者等权（默认配置）
λ = 2.0  → 语义主导
λ = 5.0  → 强语义驱动，可能损害生成质量
```

### 12.4 实验修正与方法论反思

初始消融实验存在两个方法论缺陷，导致得出了错误结论。发现和修正这些缺陷的过程本身是重要的研究经历。

#### 12.4.1 缺陷一：不公平基线

初始实验中 λ=1.0 的 checkpoint 是从预训练扩散模型续训的（等效 ~10000 步），而其余 λ 值均从零训练 5000 步。这导致 λ=1.0 的 diffusion_loss 异常低（0.18 vs 其余 ~2.0），Projector 表示质量不具可比性。

**修正**：重新从零训练 λ=1.0（5000 步），得到 diffusion_loss=2.35，与其余 λ 值一致。

#### 12.4.2 缺陷二：下游分类器过强，掩盖了上游差异

初始评估使用 2629D 特征向量（delta_z 2048D + delta_pooled 512D + 统计特征 + top-k + **23D 本体感觉信号**）+ HistGradientBoosting 分类器。结果显示所有 λ 值均达 ~87.8%（极差 < 0.7pp），呈现"不敏感"的假象。

原因分析：23D 本体感觉特征（action 7D + ee_delta 6D + 衍生特征 10D）直接包含夹爪开合度、末端执行器位移等信号，这些信号与 contact_state、gripper_state 等标签几乎是同义特征。HistGBT 作为强非线性分类器，可以仅依赖这 23D 特征就达到高精度，使得 2560D 视觉特征（delta_z + delta_pooled）的质量差异被完全掩盖。

**修正**：
1. **去除本体感觉特征**：仅使用 delta_z (2048D) + delta_pooled (512D) = 2560D 纯视觉特征
2. **添加弱分类器**：在 HistGBT 之外增加 Logistic Regression（线性分类器），进一步暴露特征空间的线性可分性差异

#### 结论：U 型曲线 — 双目标干扰效应

#### 各字段敏感性分析

| 字段 | 最低 (λ) | 最高 (λ) | 极差 | 分析 |
|------|:---:|:---:|:---:|------|
| contact_state | 60.8% (2.0) | 93.4% (0.1) | **32.6pp** | 最敏感，接触检测严重依赖 delta_z 质量 |
| gripper_state | 56.1% (2.0) | 90.4% (0.1) | **34.3pp** | 最敏感，夹爪状态与 contact 高度相关 |
| object_motion | 65.4% (1.0) | 67.7% (5.0) | 2.3pp | 不敏感，运动信息在所有 λ 条件下均被保留 |
| target_relation | 40.9% (0.5) | 43.1% (2.0/5.0) | 2.2pp | 不敏感，delta_z 不编码空间关系信息 |

#### U 型曲线的理论解释

U 型曲线的成因是扩散损失和语义损失的**梯度方向冲突**：

1. **λ 极小（0.1）**：扩散目标绝对主导，Projector 为像素重建优化，编码高保真视觉变化信息。语义信号太弱，几乎不干扰扩散学到的表示。delta_z 自然包含了接触/夹爪等像素级可见的语义。

2. **λ 中间（1.0~2.0）**：两个目标势均力敌，梯度方向不一致时 Projector 被迫在两个目标之间妥协。扩散要求编码所有像素级细节，语义要求编码分类边界，两者的最优解不重合。Projector 达成的折中表示"既不完全像素化，也不完全语义化"，导致两边都做不好。

3. **λ 极大（5.0）**：语义目标主导，Projector 直接朝着语义可分性优化。虽然牺牲了一些像素重建精度（diffusion_loss 仍在 2.14 的合理范围），但 delta_z 被显式引导编码语义相关特征，分类性能恢复。

4. **object_motion 和 target_relation 不受影响**的原因不同：
   - object_motion 的信号（物体位移）在任何表示策略下都能被保留，因为它是最显著的像素级变化
   - target_relation 的信号（与目标的空间关系）不在 delta_z 中编码，它需要绝对空间推理能力，无论 λ 如何调整都无法改善

#### 方法论教训

| 评估方案 | 均值极差 | 结论 | 问题 |
|---------|:---:|------|------|
| 2629D + HistGBT（初始） | 0.7pp | "不敏感" | 本体感觉特征掩盖了视觉特征差异 |
| 2560D + HistGBT（修正） | **16.0pp** | **U 型曲线** | 真实反映 Projector 表示质量差异 |
| 2560D + LogReg（线性探测） | 3.7pp | 弱 U 型 | 弱分类器下趋势一致但幅度减小 |

这一结果揭示了消融实验设计的关键原则：**评估下游分类器的能力必须匹配消融要隔离的变量**。当下游分类器过强（或输入特征包含直接答案）时，上游变化会被吸收，导致虚假的"不敏感"结论。

#### 消融实验的价值

1. **指导 λ 选择**：最优 λ 在两端（0.1 或 5.0），应避免中间值（1.0~2.0）。实际使用中推荐 λ=0.1（扩散主导，语义作为轻量辅助），因为它在不牺牲生成质量的同时提供了最佳语义可分性。
2. **验证多任务学习的梯度冲突理论**：U 型曲线直接支持了多目标优化中的经典现象 — 当两个目标的梯度不完全对齐时，等权组合往往是最差的选择。
3. **方法论贡献**：展示了如何通过控制变量（去除本体感觉信号、替换弱分类器）来修正初始实验的假阳性结论，这一纠错过程本身是严谨研究的体现。

## 十三、消融实验：SemanticHead 接入位置

### 13.1 实验动机

前面的实验已经证明：(1) 纯扩散训练下压缩流水线丢弃语义信息，(2) 注入语义监督可以"救回"语义。但一个关键的架构问题尚未回答：**语义信息在压缩流水线的哪一级被丢弃？每一级压缩的语义损失有多大？**

StaMo 的压缩流水线有三个层级，每一级都可能是语义信息的丢失点。通过在不同层级接入 SemanticHead，可以定量测量每一级压缩对语义保真度的影响。

### 13.2 信息流与候选接入位置

```
图像 [B, 3, 224, 224]
  │
  ▼
┌──────────────────────────────────────────────┐
│  DINOv2 VisionBackbone (frozen)              │
│  [B, 256 patches, 768D]                      │
└──────────────┬───────────────────────────────┘
               │
               │  ◀── Position C: delta_dino (768D, mean-pooled)
               │       梯度截断：DINOv2 frozen，仅训练 SemanticHead
               ▼
┌──────────────────────────────────────────────┐
│  Projector (attention + compression)         │
│  256 patches × 768D  →  2 tokens × 1024D    │
│  (trainable)                                 │
└──────────────┬───────────────────────────────┘
               │
               │  ◀── Position B: delta_proj (1024D, mean-pooled 2 tokens)
               │       梯度直达 Projector
               ▼
┌──────────────────────────────────────────────┐
│  DiTConditionHead (mean-pool + Linear)       │
│  2 tokens × 1024D  →  512D                  │
│  (trainable)                                 │
└──────────────┬───────────────────────────────┘
               │
               │  ◀── Position A: delta_pooled (512D) ★ 当前设计
               │       梯度经 DiTConditionHead → Projector
               ▼
┌──────────────────────────────────────────────┐
│  DiT 扩散模型 (条件生成)                       │
└──────────────────────────────────────────────┘
```

| 位置 | 输入维度 | 语义梯度流向 | 对应的压缩层级 |
|------|---------|-------------|--------------|
| **A: pooled** | 512D | → DiTConditionHead → Projector | 全部压缩之后（384:1） |
| **B: proj** | 1024D | → Projector（跳过最后一级压缩） | Projector 压缩之后（96:1） |
| **C: dino** | 768D | 无（DINOv2 frozen） | 压缩之前（原始特征） |

### 13.3 实验设置

- 每个位置独立从零训练 5000 步，λ=1.0
- 特征提取统一使用 delta_z (2048D) + delta_pooled (512D) = **2560D visual-only**
- 评估使用 HistGradientBoosting 和 Logistic Regression
- 这样比较的是：**训练时不同位置的语义监督，对 Projector learned representations 的影响**

### 13.4 训练损失对比

| 位置 | total_loss | diffusion_loss | semantic_loss |
|------|-----------|---------------|--------------|
| A: pooled | 8.91 | 1.80 | 7.11 |
| B: proj | 6.59 | 1.73 | **4.86** |
| C: dino | 9.99 | 2.13 | 7.86 |

关键观察：
- **diffusion_loss**：A 和 B 几乎相同（1.73~1.80），但 C 略高（2.13）。这是因为 Position C 的语义梯度无法到达 Projector（DINOv2 frozen），Projector 只被扩散损失训练，与纯扩散训练等价
- **semantic_loss**：B（4.86）远低于 A（7.11）和 C（7.86）。proj 位置的 1024D 表示包含最丰富的可分类信息。C 的 semantic_loss 最高，说明 DINOv2 原始特征虽然语义丰富（实验二证明），但 SemanticHead 从 delta_dino 中学习分类边界的效率不如从经过 Projector 训练的表示中学习

### 13.6 分析

#### 核心发现：B > A > C — 语义监督通过 Projector 放大语义编码

三个位置的排序 B (68.8%) > A (60.9%) > C (53.7%) 揭示了一个出乎意料的结论：**Projector 的 96:1 压缩不仅没有丢失语义，反而在语义监督下增强了语义编码。**

这与最初的假设相反。我们原本预期 C（DINOv2 原始特征）应该是语义最丰富的，因为它没有经过任何压缩。但实际上 C 的表现最差，甚至接近纯扩散基线（~54%）。

#### 为什么 Position C 表现最差

Position C 的语义梯度止步于 DINOv2（frozen），无法到达 Projector。这意味着：
- Projector 只被扩散损失训练，与纯扩散训练等价
- SemanticHead 只能从 DINOv2 的 delta_dino 中学习分类，但评估时用的是 Projector 输出的 delta_z + delta_pooled
- **训练信号和评估特征来自不同的表示空间**——SemanticHead 在 DINOv2 空间学到的分类能力，无法迁移到 Projector 的压缩空间

这解释了 C ≈ 纯扩散基线：语义监督存在但无法影响 Projector，等于没有语义监督。

#### 为什么 Position B 远好于 Position A

B 比 A 高 7.8pp（68.8% vs 60.9%），其中 contact_state +15.0pp，gripper_state +16.6pp。两个原因：

1. **梯度路径更短**：B 的语义梯度直接作用于 Projector 参数，A 的梯度需要经过 DiTConditionHead 间接传递。更短的梯度路径意味着更强、更精确的语义信号
2. **信息在最后一级被压缩丢弃**：DiTConditionHead 的均值池化 + 线性投影（2×1024 → 512）为扩散生成设计，会丢弃对扩散无用但对语义有用的信息。Position B 跳过了这一级，保留了更完整的语义

但两个位置的 diffusion_loss 几乎相同（1.73 vs 1.80），说明这种语义增强**不以生成质量为代价**。

#### 完整的压缩语义损失图

| 对比 | 回答的问题 | 结论 |
|------|-----------|------|
| B vs C | Projector 压缩（96:1）对语义的影响？ | **增强 +15.1pp**（语义梯度使 Projector 主动编码语义） |
| A vs B | DiTConditionHead 压缩（最后一级）的语义损失？ | **丢失 7.8pp**（均值池化丢弃语义信息） |
| A vs C | 整条流水线的净效应？ | **净增强 +7.2pp**（Projector 增强 > DiTConditionHead 丢失） |

这张图的关键洞察是：**压缩流水线对语义的影响不是单调的丢失，而是"先增强后丢失"。** Projector 在语义梯度驱动下主动编码语义（+15.1pp），但 DiTConditionHead 的最后一级压缩又丢掉了一部分（-7.8pp），净效应仍然是正的（+7.2pp）。

#### 对§1.3 核心机制的验证

这些结果精确验证了"优化目标决定压缩方向"的论点：

- **Position C**：语义梯度无法到达 Projector → Projector 只被扩散目标驱动 → 压缩丢弃语义 → 表现 ≈ 纯扩散基线
- **Position A**：语义梯度经 DiTConditionHead 间接到达 Projector → Projector 部分编码语义 → 表现中等
- **Position B**：语义梯度直达 Projector → Projector 充分编码语义 → 表现最好

**梯度到达 Projector 的效率，直接决定了压缩表示的语义质量。**

### 13.7 架构启示

1. **语义监督的最优接入点是 Projector 输出**：Position B 在所有指标上最优，且不影响生成质量。当前默认设计（Position A）并非最优选择
2. **Projector 是语义编码的关键节点**：它不是被动的信息瓶颈，而是可以被优化目标主动塑造的表示学习器。语义梯度能否有效到达 Projector，是决定压缩表示语义质量的核心因素
3. **DiTConditionHead 是语义信息的损失点**：其均值池化 + 线性投影为扩散生成设计，系统性丢弃语义信息。如果下游需要语义，应从 Projector 输出（Position B）而非 DiTConditionHead 输出（Position A）读取

## 十四、核心贡献总结

1. **负面结果（有价值）**：证明纯扩散训练下，StaMo 的压缩流水线（256×768 → 2×1024 → 512）系统性丢弃语义信息。这不是扩散模型的固有缺陷，而是**任务特异性压缩**的必然结果——当 Projector 只被扩散损失训练时，它只保留对像素重建有用的信息，语义信息因对重建贡献微弱而被丢弃。

2. **正面结果（上限参考）**：DINOv2 自监督特征 + 本体感觉信号构成强语义分类基线（contact 96.9%, gripper 91.7%, motion 99.4%）。DINOv2 的对比学习目标天然产生语义丰富的表示，证明语义信息在压缩流水线的**入口处**是充分存在的——问题出在压缩过程，而非输入。

3. **核心贡献**：提出扩散 + 语义并行训练架构，通过在压缩流水线中注入轻量级语义监督（SemanticHead，~0.6M 参数，仅占总可训练参数的 0.6%），改变 Projector 的压缩策略使其主动编码语义信息。delta_z 的语义分类性能从 ~54% 提升至 95.9%（contact_state），几乎追平 DINOv2 原始特征（96.9%），同时扩散生成质量不变（PSNR 9.10→8.96，LPIPS 1.256→1.230，差异在统计误差内）。

4. **状态抽象价值**：12 维紧凑语义向量在阶段分类（78%）、few-shot 学习（2% 数据达 91.7%）、动作预测（z 轴 R²=0.76）等下游任务中均显著优于 512 维 delta_z。信息压缩比 42.7:1 表明紧凑语义接口捕获了关键的状态转移信息。

5. **"免费午餐"验证——语义和像素信息不冲突**：生成质量评估（PSNR/SSIM/LPIPS）和延续训练实验共同证明，Projector 的 2048 维表示容量足以同时编码像素信息和语义信息。语义监督改变了压缩方向，但没有挤占像素信息的空间——以极小开销（+0.6% 参数，+10% VRAM）换取语义理解能力。

6. **压缩流水线的逐级语义损失量化**（位置消融）：通过在不同压缩层级接入语义监督，定量测量了每一级压缩丢失的语义信息。DiTConditionHead 的 1024→512 压缩丢失了 7.8pp 的语义分类准确率，但不影响扩散生成质量（diffusion_loss 一致）。这直接证明了扩散任务和语义任务的信息需求不完全重叠——为扩散优化的压缩会丢弃语义任务需要的信息。

## 十五、局限性

1. **评估方式的间接性**：所有语义分类结果都是 sklearn 分类器在提取特征上的表现，不是端到端评估。SemanticHead 在训练中只是一个梯度信号源，真正被评估的是 Projector 学到的表示质量。这种间接评估无法完全排除分类器本身引入的偏差。

2. **数据规模与多样性**：3,297 样本、单一数据集（DROID-100）、单一任务类型（桌面操作）。核心发现（优化目标决定压缩方向）在理论上应该是通用的，但实验证据的覆盖面有限。

3. **模型规模**：Toy DiT（6 层、8 头）远小于实际部署的扩散模型。更大的模型可能有更大的表示容量，"语义和像素信息不冲突"的结论在容量更紧张或更宽裕的情况下是否仍然成立，尚未验证。

4. **语义定义的局限**：4 个字段 12 个类别是手动定义的，覆盖的语义空间有限。Projector 的 2048 维表示中可能存在大量未被这 4 个字段捕获的语义结构。

5. **target_relation 瓶颈**（64.6%）：目标位置是隐含的空间概念，单帧对的 delta 表示可能缺乏足够的空间推理能力。

6. **过拟合**：SemanticHead 在 2659 训练样本上 5000 步后即开始过拟合。当前 dropout=0.1 可能不足。

## 十六、未来工作

### 16.1 从"能不能保留"到"保留了什么"——压缩表示的语义结构探测

当前实验用 4 个手动定义的语义字段来评估压缩表示的语义质量。但这只是在问"这 4 个特定概念是否被保留"，没有回答一个更根本的问题：**Projector 的 2048 维表示里，到底自发地组织出了什么语义结构？**

具体方向：

- **无监督探测**：对扩散+语义训练后的 delta_z 做聚类分析（t-SNE / UMAP 可视化，k-means），观察是否自发形成了与语义标签对齐的簇结构，以及是否存在**未被标注的语义维度**（比如"操作阶段"、"物体类型"等没有显式标签的概念）
- **对比实验**：同样的聚类分析应用于纯扩散训练的 delta_z，对比两者的结构差异。如果语义监督不仅保留了 4 个标注字段，还诱导出了更广泛的语义组织，这说明语义监督的效果比表面数字显示的更深
- **理论意义**：这将把本研究从"验证一个具体方案"推进到"理解压缩表示的语义涌现"——语义监督是否像对比学习一样，能诱导出超越监督信号本身的表示结构？

### 16.2 压缩方向的可控性——任务条件化的 Projector

当前方案用固定的 λ 和固定的语义标签集来重定向压缩。但不同下游任务对语义信息的需求不同：导航任务需要空间关系（target_relation），装配任务需要接触状态（contact_state），质检任务可能需要完全不同的语义维度。

具体方向：

- **Task-conditioned compression**：在 Projector 中引入任务条件向量，使其根据下游需求动态调整压缩策略。训练时随机采样不同的语义子集作为监督，推理时通过条件向量选择保留哪些语义
- **Adapter 方案**：保持 Projector 主干不变，为不同下游任务训练轻量级 adapter（类似 LoRA），每个 adapter 将压缩方向微调到特定语义需求
- **理论意义**：这将本研究的发现从"一个具体的修复方案"推广为"一个关于压缩可控性的框架"——信息瓶颈中保留什么信息，不再由训练时的损失函数一次性决定，而是可以在部署时按需调整

### 16.3 真实 RL 环境中的端到端验证

本研究的全部动机是"机器人操作的状态理解"，但所有实验都停留在表示质量的离线评估上。缺少的关键一环是：**语义接口是否真的能加速策略学习？**

具体方向：

- **模拟环境验证**：在 ManiSkill 或 RLBench 等标准操作基准上，对比三种策略输入的学习效率：(a) 512 维 delta_z（纯扩散表示），(b) 512 维 delta_z（扩散+语义表示），(c) 12 维语义向量。如果 (c) 以 42.7:1 的压缩比达到接近 (b) 的策略性能，这将直接验证紧凑语义接口的实用价值
- **Sample efficiency**：语义接口的核心价值假设是降低策略学习的样本复杂度。12 维离散语义空间比 512 维连续空间更容易探索，策略网络应该需要更少的交互数据。这个假设需要实验验证
- **理论意义**：这将把本研究从"表示学习的分析工作"升级为"对下游任务有实际影响的方法"，完成从"压缩会丢弃语义"到"保留语义能加速决策"的完整论证链

## 十七、RL 仿真验证实验（ManiSkill3）

### 17.1 实验动机

前述所有实验（探针、消融、下游分类）均为离线评估——它们证明了"表示里有语义"，但无法回答"有语义对策略学习有没有用"。为闭环验证语义增强表示的实际价值，我们设计了 RL 对比实验：将两组冻结的 StaMo encoder 接入相同的 PPO 策略网络，在仿真环境中训练，对比 success rate 和 sample efficiency。

### 17.2 实验平台

- **仿真环境**：ManiSkill3（GPU 并行仿真，支持多环境同时运行）
- **机器人**：Franka Panda 7-DoF 机械臂
- **控制模式**：`pd_joint_delta_pos`（8D 关节增量，动作范围 [-1, 1]）
- **RL 算法**：PPO（clipped objective, GAE）
- **硬件**：RTX 5060 8GB

### 17.3 Phase 1：ManiSkill3 数据采集

从 ManiSkill3 预录演示中回放轨迹，渲染 224×224 RGB 图像对，自动构造语义标注。

| 任务 | 图像对数 | 训练集 | 测试集 |
|------|---------|--------|--------|
| PickCube-v1 | 7,476 | 5,980 | 1,496 |
| StackCube-v1 | 10,412 | 8,329 | 2,083 |

语义标注字段：contact_state, gripper_state, object_motion, target_relation
图像对参数：step_gap=4, pair_stride=2

### 17.4 Phase 2：StaMo 表示训练（ManiSkill3 数据）

两组使用相同超参，仅 semantic_head 开关不同。

| 配置项 | 值 |
|--------|---|
| Vision backbone | DINOv2 ViT-B/14 (冻结) |
| Projector | 2 token, 4 层 cross-attention, output 1024D |
| DiT | toy mode, 6 层, 8 heads |
| batch_size | 8 |
| lr | 5e-5 (warmup + cosine decay) |
| 训练步数 | 5000 |
| GPU | RTX 5060 8GB |

#### 训练结果

| 指标 | Group A (纯扩散) | Group B (扩散+语义, λ=0.1) |
|------|---------|---------|
| Diffusion loss (5000步) | 0.20 | 0.20 |
| Semantic loss (5000步) | — | 0.29 |
| VRAM 峰值 | 4.05 GB | 5.07 GB |
| 训练速度 | 0.4 step/s | 0.4 step/s |

Group B 的 semantic loss 从 1.06 (初始) → 0.53 (1000步) → 0.29 (5000步)，持续下降。

#### 探针验证（5000 步 checkpoint）

**线性探针**（Logistic Regression on delta_pooled）：

| 语义字段 | Group A | Group B | 差异 |
|----------|---------|---------|------|
| contact_state | 50.8% | 50.3% | -0.4% |
| gripper_state | 47.2% | 56.6% | +9.4% |
| object_motion | 53.2% | 58.4% | +5.1% |
| target_relation | 49.7% | 56.1% | +6.5% |
| **MEAN** | **50.2%** | **55.3%** | **+5.1%** |

**MLP 探针**：

| 语义字段 | Group A | Group B | 差异 |
|----------|---------|---------|------|
| **MEAN** | **59.6%** | **85.6%** | **+26.0%** |

Group A 的 pooled 表示接近随机水平（~50%），确认纯扩散目标不编码语义。Group B 在 MLP 探针下达到 85.6%，确认语义信息以非线性方式被编码进表示中。

### 17.5 Phase 3a：RL 基线验证

在接入 StaMo encoder 之前，先验证 PPO 能否在纯状态观测下学会任务，排除 RL 算法本身的问题。

#### 控制模式探索

| 控制模式 | 动作空间 | 结果 |
|----------|---------|------|
| `pd_joint_pos` | 8D 绝对关节位置 | ❌ 500K 步 success=0% |
| `pd_ee_delta_pos` | 4D 末端增量 | ❌ Windows 无法安装 pinocchio |
| `pd_joint_delta_pos` | 8D 关节增量 | ✅ 采用 |

#### 环境并行化

实现了 64 环境 GPU 并行，采样速度 ~1100 steps/s。过程中解决了两个关键问题：
1. **sapien `cuda.dll` bug**：sapien 在 Windows 上错误加载 `cuda.dll`（应为 `nvcuda.dll`），手动修复
2. **GPU 模式不自动重置**：需手动调用 `env.reset(options={'env_idx': done_idx})`

#### 任务选择

| 任务 | 配置 | 结果 | 结论 |
|------|------|------|------|
| PickCube-v1 | 64 并行, 1M 步 | 0.1% success (8/10880 episodes) | 太难，多阶段操作 vanilla PPO 学不会 |
| PushCube-v1 | 64 并行, 64K 步 | 0.2% success (1/640) | 合适，单阶段推动任务 |

PickCube-v1 + StaMo encoder 的完整实验（500K 步）也确认了 0% success rate，return 在 ~1.0 平台期。

**选择 PushCube-v1 的理由**：
- PPO 可学（64K 步即出现 success）
- 任务成功依赖物体运动方向和目标关系的感知，与语义字段直接相关
- 单阶段任务，排除多阶段序列带来的额外难度

### 17.6 Phase 3b：StaMo Encoder + PPO 对比实验 🔄 进行中

#### 实验方案

```
RGB 观测 → StaMo encoder (冻结) → 2560D 视觉特征 + 本体感知 → MLP(256,256) → 动作
```

| 配置项 | 值 |
|--------|---|
| 任务 | PushCube-v1 |
| 对比组 | Group A (纯扩散) vs Group B (扩散+语义, λ=0.1) |
| 控制模式 | `pd_joint_delta_pos` |
| 算法 | PPO (clip=0.2, γ=0.99, λ_GAE=0.95) |
| 策略网络 | Actor-Critic MLP(256, 256) |
| 并行环境 | 单环境（StaMo encoder 需渲染 RGB，不兼容 GPU 并行） |
| 总步数 | 500K timesteps |
| 种子 | 42, 123, 456 |
| 评估频率 | 每 10K 步，50 episodes |

#### 评估指标

1. **Success rate**（主指标）：最终成功率对比
2. **Episode return**（辅助指标）：反映表示质量
3. **Sample efficiency**：达到特定 success rate 所需步数
4. **统计显著性**：3 seed 均值 ± std

#### 预计耗时

6 runs × 4-8h = 24-48h（单环境 + StaMo encoder 推理）

#### 预期结论

若 Group B 的 success rate 更高或收敛更快，则闭环证明：语义增强的表示对策略学习有实际帮助——完成从"压缩会丢弃语义"到"保留语义能加速决策"的因果链。

### 17.7 Phase 4：后续分析（待完成）

1. **学习曲线对比**：success rate vs timesteps，两组 ± std
2. **统计检验**：3 seed 的均值差异是否显著
3. **扩展到 StackCube**：多阶段操作任务，语义变化更丰富，预期差异更明显
4. **尝试更强 RL 算法**（SAC 等）：排除"仅在弱算法上有效"的可能
5. **Sim-to-real 迁移探索**：语义表示理论上对外观变化更鲁棒

### 17.8 关键技术发现

1. **控制模式选择至关重要**：`pd_joint_pos`（绝对位置）对 RL 极不友好，`pd_joint_delta_pos`（增量）效果好得多
2. **ManiSkill3 GPU 并行需要手动重置**：与文档描述不同，GPU 模式下 episode 结束后不会自动 reset
3. **sapien Windows 兼容性 bug**：`cuda.dll` 应为 `nvcuda.dll`
4. **PickCube 对 vanilla PPO 过难**：多阶段操作任务需要更强算法或更多训练步数
5. **StaMo encoder 推理限制并行度**：接入 encoder 后无法使用 GPU 并行环境，回退到单环境模式
